Direct Mapping

Each memory block maps to exactly one cache line. Lookup is instant — just compute the index. However, conflict misses occur when two frequently used blocks share the same slot.

index = block_addr mod cache_size
Fast lookupConflict missesLow cost
🔀

Fully Associative

Any memory block can go into any cache line. Uses LRU for replacement. Maximum flexibility with zero conflict misses, but requires parallel search hardware.

Search all lines; replace LRU on miss
No conflictsHigh costMax flexibility
⚖️

Set Associative

Hybrid approach — cache divided into sets; each block maps to one set, placed freely within that set (LRU). 2-way and 4-way are the industry standard used in modern CPUs.

set = addr mod num_sets; LRU in set
Best balanceIndustry standardModerate cost

Performance Metrics

Three key measurements tell us how well a cache is performing. AMAT combines both the hit and miss cases into one average access cost.

AMAT = Tc + miss_ratio × Tm
Hit Ratio = H/NMiss Ratio = 1 − h

Full Formula Reference

Hit Ratioh = Total Hits / Total Accesses
Miss Ratiom = 1 − h
AMAT= h × Tc + m × (Tc + Tm)
Simplified= Tc + m × Tm
TcCache access time (ns)
TmMain memory time (ns)
